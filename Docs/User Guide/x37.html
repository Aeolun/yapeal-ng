2.2. Editing configuration file

Yapeal expect to find a file named yapeal.ini in the config directory. There is an example configuration file in the config directory of Yapeal named yapeal.ini-example that you'll want to copy and rename to yapeal.ini then edit it for your local environment. The example have been largely commented to help users with any changes needed.

The default configuration for Yapeal is to use a separate database for each of the areas char, corp, eve, util but if you are limited on the number of databases you can have or want to use you can combine some or all of the different areas into a single database. Combining char and corp is the most likely area to cause problems because of overlapping table names.

Though Yapeal was made with the idea of having two username/password pairs per database to separate select only (reader) access and select, insert, update (writer) access that has been mostly unused and you only need one usename/password pair with full write access to each database and the same pair can and normally is used for all databases in a multiple database setup. For future versions all database access will only use the writer and all the reader settings will be removed but are being kept for now for backwards compatability with my other projects.
