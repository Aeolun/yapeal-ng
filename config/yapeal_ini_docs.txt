YAPEAL INI File

REQUIRED Sections: API, DATABASE, TRACING, PATHS, LOGGING
;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;
[API]
     This section contains settings relating to the API data files pulled
     from the API server. It must contain the following items

        cache_xml=FALSE
           Turns on caching of API XML data to local files. This can be
           either TRUE or FALSE

        file_suffix=".xml.aspx"
           File suffix of API files recieved from the CCP API server this
           should only change if CCP changes the output file names or you
           are using a non-standard source for the API data files.

        url_base="http://API.eve-online.com"
           This is the base url of the CCP API server and should only be
           changed if CCP changes the URL of the API server or if you
           are using a non-standard source for the API data files.

[DATABASE]
	* YAPEAL is designed to use multiple databases to contain the data pulled from the api but
	* can also be used with a single database.
	* 	[DATABASE]
	* 		[DATABASE-CHAR]
	* 		[DATABASE-CORP]
	* 		[DATABASE-EVE]
	* 		[DATABASE-SERVER]
	* 		[DATABASE-UTIL]

	* The simplest way to get YAPEAL working is to simply set all the values in
	  the main [DATABASE] section to those appropriate to your YAPEAL installation.

	* Then ensure the sections for the specific sub-databases are commented out.
	  YAPEAL will then use this single database for all its storage needs.

	* While this is is the simplest approach it is not the best. There may be
	  conflicts between personal and corporate data due to duplication of field
		names in some of the API data.

	* The more complex but probably more robust solution would be to create each
	  of the sub-databases and set the values in each sub-database section to
		those appropriate to your YAPEAL installation. Set the "active" flag in the
		main database section to FALSE. This option also allows you to pick and
		choose which API data will be pulled. Simply comment out or set the "active"
		flag to FALSE for each section you do not wish to poll.

	active=FALSE
		Determines if pulls for this database type are active. Since this section
		acts as the default for all database activity, setting this to TRUE will
		cause all API data to be pulled and is the same as setting each of the
		sub-database "active" fields to true.
		Don't set this to TRUE except if you wish to have all Database-xxx sections
		on by default and plan to set unused sections off below.

	database="YAPEAL"
		Main Database name
		Database name used if only using a single database or the default database
		name used if not overridden in a [database-xxx] section below.

	host="localhost"
		Host name for ALL databases - if set here it need not be set in each
		database-xxx section.

	writer="writer:5678"
		user:password with write access to ALL databases . (required)
		Needs minimum of select, insert, and update access.
		The :password part is optional by highly recommended.

	STUFF YOU WONT WANT TO CHANGE UNLESS YOU KNOW YOU WANT TO.
	driver="mysqli://"
		Driver part of ADOdb DSN. Must support transactions.
		no need to change this unless you know you need to change it.

	suffix="?new"
		ADOdb needs a setting of "?new" to work correctly with MySQL.

[DATABASE-CHAR]
	Main database where YAPEAL stores character API data. (required for chars)

	active=FALSE
		Determines if pulls for this database type are active. (required)
		Set to FALSE to turn off pulls.

	database="YAPEAL_char"
		Database name used in ADOdb DSN.

	writer="writer:5678"
		User/password with select, insert, and update access.

[LOGGING]
	Logging settings (required)

	error_log=YAPEAL_error.log
		Error level log file name. (required)

	level=E_USER_ERROR|E_USER_WARNING|E_USER_NOTICE
		Logging levels which are enabled. (required)
		Should be set like error_reporting() but ONLY use E_USER_* values

	notice_log=YAPEAL_notice.log
		Notice level log file name. (required)

	trace_log=YAPEAL_trace.log
		Trace level log file name. (required)
		This is only used if tracing is turned on in [Tracing]

	warning_log=YAPEAL_warning.log
		Warning level log file name. (required)

[Paths]
	Path settings used by YAPEAL. (required)

	adodb_dir="../ADOdb/"
		File system path to ADOdb directory. (required)
		Should be relative to common_backend.inc's path.

	base_dir="../"
		File system path to base of YAPEAL directory. (required)
		Should be relative to common_backend.inc's path.

	backend_dir="../backend/"
		File system path to backend files directory. (required)
		Should be relative to common_backend.inc's path.

	cache_dir="../cache/"
		File system path to cache directory. (required)
		Should be relative to common_backend.inc's path.
		Must be writable.

	class_dir="../class/"
		File system path to class directory. (required)
		Should be relative to common_backend.inc's path.

	config_dir="../config/"
		File system path to config directory. (required)
		Should be relative to common_backend.inc's path.

	inc_dir="./"
		INC directory path. (required)
		Should be relative to common_backend.inc's path.
		DEBUGGING ONLY This should never be change in production.

	log_dir="log/"
		File system path to logging directory. (required)
		Must be writable.
		NOTE Should be relative to cache_dir's path.
;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;;
[Tracing]
	Settings used when debugging YAPEAL. (required)

	trace=FALSE
		Set to TRUE to turn tracing on. (required)

	trace_level=0
		Sets level of debugging wanted. (required)
		Used in a few places to turn on extra tracing when >0.

	trace_output="file"
		Where you want trace output to go. (required)
		One of "file", "database", or "both"
		NOTE database option isn't active but is being reserved for the future.

	trace_section=YAPEAL_TRACE_NONE
		Sections that you wish to log trace messages for. (required)
		Set to YAPEAL_TRACE_ALL or YAPEAL_TRACE_NONE or
		YAPEAL_TRACE_ACCOUNT, YAPEAL_TRACE_CHAR, YAPEAL_TRACE_CORP, YAPEAL_TRACE_EVE,
		YAPEAL_TRACE_MAP, YAPEAL_TRACE_SERVER,
		YAPEAL_TRACE_API, YAPEAL_TRACE_CACHE, YAPEAL_TRACE_CURL, YAPEAL_TRACE_DATABASE,
		YAPEAL_TRACE_FILES, YAPEAL_TRACE_REQUEST
		Settings can be combined with '|' for example:
			trace_section=YAPEAL_TRACE_EVE|YAPEAL_TRACE_REQUEST
